Python Tutorial: Working with Files 📁

Welcome to our Python journey! Today we'll learn about working with files - a super important skill for any programmer. We'll learn how to read and write files safely, and create some cool projects!

🛡️ File Safety First!

Before we start our file adventure, let's learn how to stay safe when working with files on your computer.

# 🚫 DON'T work in these folders:
"C:\Windows"      # System folders
"C:\Program Files"# Program folders
"/usr/bin"        # Linux system folders
"/System"         # Mac system folders

# ✅ DO work in these folders:
"Documents/python_practice"  # Your practice folder
"Desktop/python_files"      # A folder you create

Why These Safety Rules?

  1. Stay in Your Space:

    • Think of your computer like a house
    • System folders are like your parents' room - don't mess with them!
    • Make your own "room" (folder) for Python practice

  2. Always Make Backups:

    • Before changing any file, make a copy
    • Name it something like original_file_backup.txt
    • If something goes wrong, you have the original!

  3. Start Small:

    • Begin with tiny text files (a few lines)
    • Use test files you create yourself
    • Don't experiment with important files

1. Understanding Files and Folders 📂

What's a File Path?

Think of a file path like giving directions to your house:

Country
  └── City
       └── Street
            └── House Number

On your computer, it's similar:

C:\
  └── Documents
       └── python_practice
            └── my_file.txt

Types of Paths:

# Absolute Path (full address)
full_path = "C:/Documents/python_practice/my_file.txt"

# Relative Path (directions from where you are)
relative_path = "./my_file.txt"  # Current folder
relative_path = "../my_file.txt" # Parent folder

2. Reading Files 📖

The Safe Way to Open Files

Always use with statements - they automatically close your file when you're done:

# ✅ The GOOD way:
with open("my_diary.txt", "r") as file:
    content = file.read()
    # File closes automatically when done!

# ❌ The BAD way:
file = open("my_diary.txt", "r")
content = file.read()
# Oops! Forgot to close the file!

Reading Examples:

# Example 1: Reading a simple diary
with open("my_diary.txt", "r") as diary:
    # Read the whole file at once
    all_entries = diary.read()
    print("My diary contains:", all_entries)

# Example 2: Reading line by line
with open("my_diary.txt", "r") as diary:
    # Read one line at a time
    for line in diary:
        print("Today's entry:", line)

# Example 3: Reading into a list
with open("high_scores.txt", "r") as scores:
    # Get all lines as a list
    score_list = scores.readlines()
    print("Top score:", score_list[0])

3. Writing Files ✍️

Writing to files is like writing in a notebook - you can either start with a fresh page (write mode) or add to the end of existing notes (append mode).

Creating and Writing Files

We use different modes when writing files:

  • "w" - Write mode: Creates a new file or overwrites an existing one
  • "a" - Append mode: Adds content to the end of an existing file
# Example 1: Writing a new file
with open("shopping.txt", "w") as file:
    file.write("Apples\n")  # \n means "new line"
    file.write("Bananas\n")
    file.write("Oranges\n")

# Example 2: Appending to a file (adding more items)
with open("shopping.txt", "a") as file:
    file.write("Milk\n")    # Adds to end of file

# Example 3: Writing multiple lines at once
games = ["Minecraft", "Roblox", "Pokemon"]
with open("games.txt", "w") as file:
    file.writelines(game + "\n" for game in games)

🚨 Important Safety Tips!

Just like keeping backup copies of your homework, it's important to backup files before changing them:

# Before overwriting a file, check if it exists
import os

filename = "important_notes.txt"
if os.path.exists(filename):
    # Make a backup first
    backup_name = filename + ".backup"
    with open(filename, "r") as source:
        with open(backup_name, "w") as backup:
            backup.write(source.read())
    print(f"Backup created: {backup_name}")

# Now it's safe to write the file
with open(filename, "w") as file:
    file.write("New content\n")

4. Working with CSV Files 📊

CSV files are like digital spreadsheets - perfect for storing information in rows and columns. Think of them as Excel files in their simplest form.

Reading CSV Files

import csv

# Example 1: Reading a simple CSV file
# grades.csv contains: Subject,Grade
#                     Math,A
#                     Science,B+
with open("grades.csv", "r") as file:
    csv_reader = csv.reader(file)
    for row in csv_reader:
        print(f"Subject: {row[0]}, Grade: {row[1]}")

# Example 2: Reading CSV with headers
# students.csv has column names in first row
with open("students.csv", "r") as file:
    csv_reader = csv.DictReader(file)
    for row in csv_reader:
        print(f"Name: {row['name']}, Age: {row['age']}")

Writing CSV Files

# Example 1: Writing a simple class list
data = [
    ["Name", "Age", "Grade"],  # Header row
    ["Alice", "12", "A"],
    ["Bob", "13", "B"],
    ["Charlie", "12", "A"]
]

with open("class.csv", "w", newline='') as file:
    csv_writer = csv.writer(file)
    csv_writer.writerows(data)

# Example 2: Writing student records using dictionaries
students = [
    {"name": "David", "age": 13, "grade": "B+"},
    {"name": "Emma", "age": 12, "grade": "A-"}
]

with open("more_students.csv", "w", newline='') as file:
    fieldnames = ["name", "age", "grade"]
    writer = csv.DictWriter(file, fieldnames=fieldnames)
    
    writer.writeheader()  # Write the column names
    writer.writerows(students)

🎮 Mini-Project: Grade Tracker

Let's build a program that helps teachers keep track of student grades. It will store grades in a CSV file and let us look up how any student is doing.

import csv

def add_grade(filename, student, subject, grade):
    # Adds a new grade entry to our grade book
    with open(filename, "a", newline='') as file:
        writer = csv.writer(file)
        writer.writerow([student, subject, grade])

def get_student_grades(filename, student_name):
    # Finds and returns all grades for a student
    grades = []
    with open(filename, "r") as file:
        reader = csv.reader(file)
        for row in reader:
            if row[0] == student_name:
                grades.append(f"{row[1]}: {row[2]}")
    return grades

# Example usage:
print("=== School Grade Tracker ===")

# Record some grades
add_grade("grades.csv", "Alice", "Math", "A")
add_grade("grades.csv", "Alice", "Science", "B+")

# Check Alice's progress
print("\nAlice's grades:")
for grade in get_student_grades("grades.csv", "Alice"):
    print(grade)

5. Understanding Errors and Handling Them 🛠️

What Are Errors?

Think of errors like warning messages from Python. Just like a parent telling you "you can't do that!", Python tells us when something's not right.

# Common file errors you might see:
file = open("missing_file.txt")  # FileNotFoundError
# Python says: "Hey! This file doesn't exist!"

number = int("hello")  # ValueError
# Python says: "I can't turn 'hello' into a number!"

Introducing try-except: Your Error Safety Net 🎯

Think of try-except like a safety net at a circus:

  • The acrobat (your code) tries to do something
  • If they fall (an error occurs), the net (except) catches them
# Without try-except (dangerous!)
file = open("data.txt")           # Program crashes if file missing!

# With try-except (safe!)
try:
    file = open("data.txt")
    content = file.read()
    print("File content:", content)
except FileNotFoundError:
    print("Oops! The file wasn't found. Check the file name!")

Common File Errors and How to Handle Them

# Example 1: Missing File
try:
    with open("homework.txt") as file:
        content = file.read()
except FileNotFoundError:
    print("Can't find homework.txt - did you create it?")

# Example 2: Permission Error (trying to access a locked file)
try:
    with open("secret_file.txt") as file:
        content = file.read()
except PermissionError:
    print("Sorry, you're not allowed to read this file!")

# Example 3: Multiple Error Types
try:
    with open("notes.txt") as file:
        content = file.read()
        numbers = [int(x) for x in content.split()]
except FileNotFoundError:
    print("File is missing!")
except ValueError:
    print("File doesn't contain valid numbers!")

👉 Practice: Safe File Operations

Let's create a safe file reader that handles different problems:

def safe_read_file(filename):
    """
    Safely reads a file and handles possible errors.
    Returns the content if successful, or None if there's an error.
    """
    try:
        with open(filename, 'r') as file:
            return file.read()
    except FileNotFoundError:
        print(f"Sorry, {filename} doesn't exist!")
    except PermissionError:
        print(f"Not allowed to read {filename}!")
    except Exception as e:
        print(f"Something went wrong: {e}")
    
    return None

# Try it out!
content = safe_read_file("my_diary.txt")
if content:
    print("Successfully read the file!")
    print(content)

Advanced Error Handling 🌟

Sometimes we want to check for problems before they happen:

import os

def safe_write_file(filename, content):
    """
    Safely writes to a file with checks and backups.
    """
    # Check 1: Does the folder exist?
    folder = os.path.dirname(filename)
    if folder and not os.path.exists(folder):
        print(f"Creating folder: {folder}")
        os.makedirs(folder)
    
    # Check 2: File already exists? Make backup!
    if os.path.exists(filename):
        backup_name = filename + '.backup'
        try:
            with open(filename, 'r') as source:
                with open(backup_name, 'w') as backup:
                    backup.write(source.read())
            print(f"Backup created: {backup_name}")
        except Exception as e:
            print(f"Couldn't create backup: {e}")
            return False
    
    # Now try to write the file
    try:
        with open(filename, 'w') as file:
            file.write(content)
        print("File written successfully!")
        return True
    except Exception as e:
        print(f"Error writing file: {e}")
        return False

# Try it out:
success = safe_write_file("notes/diary.txt", "Dear Diary...")

🎮 Mini-Project: Safe Grade Manager

Let's update our grade tracker to handle errors properly:

import csv
import os

def safe_add_grade(filename, student, subject, grade):
    """
    Safely adds a grade to the CSV file.
    """
    try:
        # Check if grade is valid (A, B, C, D, F)
        if grade not in ['A', 'B', 'C', 'D', 'F']:
            print("Invalid grade! Use A, B, C, D, or F")
            return False
            
        # Create file if it doesn't exist
        if not os.path.exists(filename):
            with open(filename, 'w', newline='') as file:
                writer = csv.writer(file)
                writer.writerow(['Student', 'Subject', 'Grade'])
        
        # Add the grade
        with open(filename, 'a', newline='') as file:
            writer = csv.writer(file)
            writer.writerow([student, subject, grade])
        print(f"Added grade {grade} for {student} in {subject}")
        return True
        
    except PermissionError:
        print("Can't write to file - check permissions!")
    except Exception as e:
        print(f"An error occurred: {e}")
    return False

# Test it out
safe_add_grade("grades.csv", "Alice", "Math", "A")    # Should work
safe_add_grade("grades.csv", "Bob", "Science", "Z")   # Invalid grade
