Tutorial 7: Working with Files - Read, Write, and Manage Data 📂

1️⃣ Why Work with Files in Python?

Files allow us to store and retrieve data for later use. Instead of losing information when a program ends, we can save it to a file.

✅ Why Files? - Store data permanently - Share and manage structured information - Work with logs, user data, and reports

2️⃣ Reading from a File 📖

Opening and Reading a File


with open("example.txt", "r") as file:
    content = file.read()
    print(content)
✅ Explanation: - open("filename", "r") opens the file in read mode. - .read() retrieves the entire content. - with open(...) ensures the file closes automatically after use.

Reading Line by Line


with open("example.txt", "r") as file:
    for line in file:
        print(line.strip())  # Removes extra newlines
✅ Try-it: Modify this to count the number of lines in a file.

3️⃣ Writing to a File ✍️

Writing and Overwriting a File


with open("example.txt", "w") as file:
    file.write("Hello, Python learners!\n")
✅ Explanation: - "w" mode overwrites existing content. - write() adds new text.

Appending to a File


with open("example.txt", "a") as file:
    file.write("Appending this line!\n")
✅ Use a mode to add new data without deleting existing content.

4️⃣ Working with CSV Files 📊

CSV (Comma-Separated Values) files store structured data like spreadsheets.

Reading a CSV File


import csv
with open("data.csv", "r") as file:
    reader = csv.reader(file)
    for row in reader:
        print(row)
✅ Explanation: - The csv.reader() processes each line as a list of values. - Each row is printed as a list.

Writing to a CSV File


import csv
data = [["Name", "Age"], ["Alice", 25], ["Bob", 30]]
with open("data.csv", "w", newline="") as file:
    writer = csv.writer(file)
    writer.writerows(data)
✅ Try-it: Modify the program to append instead of overwrite.

5️⃣ Exception Handling in File Operations ⚠️

Handling Missing Files Gracefully


try:
    with open("nonexistent.txt", "r") as file:
        content = file.read()
except FileNotFoundError:
    print("Error: File not found!")
✅ Try-it: Modify this to ask the user for a filename and handle errors properly.
