Tutorial 5: Test and Fix Your Code - Debugging & Problem Solving 🔍

1️⃣ Introduction: Debugging as a Learning Tool

Debugging is more than just fixing errors—it’s a way to solidify your understanding of programming. The process of finding and fixing bugs forces you to think critically, apply your knowledge, and become a better problem solver.

✅ Why Debugging is Important? - Reinforces programming concepts. - Helps identify gaps in understanding. - Develops problem-solving skills.

💡 Your Goal: Instead of just fixing bugs, reflect on why they happen and how to prevent them.

2️⃣ Common Types of Bugs in Python ⚠️

1. Syntax Errors (Breaking the Rules)

Occurs when Python doesn’t understand your code.


print("Hello"  # ❌ Missing closing parenthesis

How to fix it? - Read the error message carefully. - Check for missing colons, parentheses, or indentation issues.

2. Runtime Errors (Crashes While Running)

Python understands your code but encounters an issue while executing.


num = int("hello")  # ❌ ValueError: invalid literal for int()
print(10 / 0)        # ❌ ZeroDivisionError: division by zero

How to fix it? - Validate user inputs. - Use try-except blocks to handle errors gracefully.

3. Logic Errors (Wrong Results)

Your program runs but produces incorrect output.


x = 10
y = 5
print("Sum:", x - y)  # ❌ Incorrect logic! Should be x + y

How to fix it? - Use print statements to track variable values. - Walk through the logic manually.

3️⃣ Debugging Techniques 🛠️

1. Print Statements: Your First Debugging Tool


def calculate_total(price, tax):
    print("Debug: price =", price, "tax =", tax)  # Print values before calculations
    total = price * tax  # ❌ Incorrect logic!
    print("Debug: total =", total)
    return total
calculate_total(100, 0.1)
✅ Fix: total = price + (price * tax)

2. Using assert to Validate Assumptions


def divide(a, b):
    assert b != 0, "Error: Division by zero!"
    return a / b
print(divide(10, 2))  # ✅ Works fine
print(divide(5, 0))   # ❌ AssertionError: Division by zero!

3. Try-Except for Handling Errors


try:
    num = int(input("Enter a number: "))
    print("You entered:", num)
except ValueError:
    print("Invalid input! Please enter a valid number.")

amount = int(input("Enter amount to withdraw: "))
if amount > balance:
    print("Error: Not enough balance!")
else:
    balance = balance - amount  # Fix logical errors here
    print("Transaction successful! New balance:", balance)

✅ Your Task: Identify and fix errors so the ATM behaves correctly.

2. Prime Number Detector 🔢


def is_prime(n):
    if n < 2:
        return False
    for i in range(2, n):
        if n % i == 0:
            return False
    return True
print(is_prime(11))  # Expected output: True
print(is_prime(4))   # Expected output: False
✅ Your Task: Add print statements to explain each step.

3. FizzBuzz Debugging Challenge 🏆


for num in range(1, 51):
    if num % 3 == 0:
        print("Fizz")
    elif num % 5 == 0:
        print("Buzz")
    elif num % 3 == 0 and num % 5 == 0:
        print("FizzBuzz")  # ❌ Bug: This condition never executes!
    else:
        print(num)
✅ Your Task: Fix the logic so that multiples of both 3 and 5 print "FizzBuzz".
