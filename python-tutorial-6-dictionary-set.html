1. Dictionaries: Magical Key-Value Pairs üìö

Dictionary Visualization:

-------------------

"age"   ->     15

A dictionary maps keys to values. Think of it like your school locker system:

  • Each locker (key) has a unique number
  • Inside each locker (value) is your stuff
  • You can't have two lockers with the same number
  • But different lockers can contain similar items

Creating and Accessing Dictionaries

1. Creating a Dictionary

# Mapping fruits to their prices
fruit_prices = {
    "apple": 2.5,
    "banana": 1.2,

3. Accessing Values

print(fruit_prices["apple"])  # Output: 2.5

# üö® Common Mistake Alert! üö®
# This will cause an error if the key doesn't exist:
# print(fruit_prices["mango"])  # KeyError: 'mango'

# ‚úÖ Better Way: Use .get() to avoid errors

2. Sets: Unique Collection Magic ‚ú®

Set Visualization:
{üçé, üçå, üçí}
No duplicates allowed!
Order doesn't matter!

Sets are like magical bags that only keep unique items. Perfect for:

  • Removing duplicates
  • Checking if something exists
  • Comparing collections

Creating and Using Sets

# Creating a set
fruits = {"apple", "banana", "cherry"}

# Creating an empty set
empty_set = set()  # Don't use {} - that creates an empty dictionary!

Need Help? ü§î

Common mistakes to avoid:

  1. Using square brackets [] to access dictionary keys that might not exist
  2. Trying to create an empty set with {}
  3. Forgetting that sets don't maintain order
  4. Using mutable types (like lists) as dictionary keys
