Tutorial 11: Error Handling & Defensive Programming ⚠️🔍

1️⃣ Why Learn Error Handling?

Programs often encounter errors due to unexpected inputs, missing files, or logic mistakes. Instead of letting a program crash, we can handle errors gracefully and make our code more robust.

✅ Why Error Handling? - Prevents program crashes. - Provides user-friendly error messages. - Helps debug and track issues efficiently.

🔍 In this tutorial, you’ll learn: - How to handle different types of errors using try-except. - How to raise custom exceptions. - Defensive programming techniques to prevent errors.

2️⃣ Handling Errors with Try-Except 🔄

The try-except block allows us to catch errors and handle them gracefully.


try:
    num = int(input("Enter a number: "))
    print(f"You entered: {num}")
except ValueError:
    print("Invalid input! Please enter a valid number.")
✅ Try-it: Modify this to keep asking the user until they enter a valid number.

Handling Multiple Exceptions


try:
    x = 10 / 0  # Division by zero error
except ZeroDivisionError:
    print("Error: Cannot divide by zero!")
except ValueError:
    print("Invalid value provided!")
✅ Mini Challenge: Extend this to handle KeyboardInterrupt (when the user stops the program).

3️⃣ Raising Custom Exceptions 🚨

You can create custom error messages using raise.


def withdraw(amount):
    if amount < 0:
        raise ValueError("Withdrawal amount cannot be negative!")
    print(f"Withdrawing ${amount}")
try:
    withdraw(-100)
except ValueError as e:
    print(f"Error: {e}")
✅ Try-it: Modify this to prevent withdrawing more money than available in an account.

4️⃣ Defensive Programming Techniques 🛡️

Defensive programming helps prevent errors before they happen.

1. Input Validation


def get_positive_number():
    while True:
        try:
            num = int(input("Enter a positive number: "))
            if num > 0:
                return num
            else:
                print("Please enter a positive number!")
        except ValueError:
            print("Invalid input! Enter a valid number.")
print(get_positive_number())
✅ Challenge: Modify this to ensure the number is within a given range.

2. Checking File Existence Before Opening 📂


import os
filename = "data.txt"
if os.path.exists(filename):
    with open(filename, "r") as file:
        print(file.read())
else:
    print("Error: File does not exist!")
✅ Try-it: Modify this to prompt the user for a filename and check if it exists before opening.

5️⃣ Logging Errors for Debugging 📝

Instead of printing errors, logging helps track and debug issues efficiently.

Using the Logging Module


import logging
logging.basicConfig(filename="app.log", level=logging.ERROR)
try:
    x = 1 / 0
except ZeroDivisionError as e:
    logging.error(f"Error occurred: {e}")
    print("An error was logged!")
✅ Try-it: Modify this to log different types of errors and check the generated log file.
